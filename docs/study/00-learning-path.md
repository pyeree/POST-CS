# RAG + MCP + AI Agent 학습 로드맵

> **목표:** Post-CS 프로젝트에 RAG와 MCP 에이전트를 도입하기 위한 기초 개념 학습
> **대상:** RAG/MCP/에이전트 개념이 처음인 개발자
> **작성일:** 2026-02-12

---

## 학습 순서

아래 순서대로 학습하면 각 개념이 자연스럽게 쌓인다.

```
[1] 임베딩과 벡터       ← 모든 것의 기초. "텍스트를 숫자로 바꾸는 법"
      ↓
[2] 벡터 데이터베이스   ← 벡터를 저장하고 검색하는 방법
      ↓
[3] RAG 파이프라인      ← 검색 + 생성을 결합하는 패턴
      ↓
[4] MCP 프로토콜        ← LLM이 외부 도구를 쓰는 표준 방식
      ↓
[5] AI 에이전트 기초    ← LLM이 스스로 판단하고 행동하는 구조
      ↓
[6] Agentic RAG         ← 에이전트 + RAG를 결합한 최종 형태
```

## 각 단계별 학습 포인트

### 01. 임베딩과 벡터 (`01-embeddings-and-vectors.md`)
- **핵심 질문:** "컴퓨터가 텍스트의 '의미'를 어떻게 이해하는가?"
- **배우는 것:** 임베딩이란, 벡터 공간, 코사인 유사도, 임베딩 모델
- **Post-CS 연결:** 블로그 글을 벡터로 변환해서 유사한 글을 찾는 기반

### 02. 벡터 데이터베이스 (`02-vector-databases.md`)
- **핵심 질문:** "수만 개의 벡터에서 비슷한 것을 어떻게 빠르게 찾는가?"
- **배우는 것:** 벡터 DB의 역할, ChromaDB 사용법, 인덱싱, 메타데이터 필터링
- **Post-CS 연결:** `src/memory/vector_store.py`의 기반 기술

### 03. RAG 파이프라인 (`03-rag-pipeline.md`)
- **핵심 질문:** "LLM이 모르는 정보를 어떻게 알려줄 수 있는가?"
- **배우는 것:** RAG 아키텍처, 청킹 전략, 검색 패턴, 하이브리드 검색, 리랭킹
- **Post-CS 연결:** 기존 블로그 글을 context로 넣어서 더 나은 글 생성

### 04. MCP 프로토콜 (`04-mcp-protocol.md`)
- **핵심 질문:** "LLM이 외부 시스템을 어떻게 조작하는가?"
- **배우는 것:** MCP 아키텍처, 서버/클라이언트, Tool/Resource/Prompt, 통신 방식
- **Post-CS 연결:** Notion, Velog, ChromaDB를 MCP 서버로 노출

### 05. AI 에이전트 기초 (`05-ai-agents.md`)
- **핵심 질문:** "LLM이 스스로 계획하고 실행하려면 무엇이 필요한가?"
- **배우는 것:** 에이전트 루프, ReAct 패턴, 도구 호출, 메모리, 계획 수립
- **Post-CS 연결:** 블로그 파이프라인을 자율적으로 실행하는 에이전트

### 06. Agentic RAG (`06-agentic-rag.md`)
- **핵심 질문:** "에이전트가 검색을 '똑똑하게' 하려면?"
- **배우는 것:** 기본 RAG의 한계, Agentic RAG 패턴, 쿼리 재작성, 자기 평가
- **Post-CS 연결:** 에이전트가 검색 품질을 판단하고 재검색하는 구조

## 학습 팁

1. **각 문서를 순서대로 읽되, 코드 예제는 직접 실행해볼 것**
2. **"이게 내 프로젝트에서 어디에 쓰이지?"를 항상 생각할 것**
3. **완벽히 이해하고 넘어가려 하지 말 것** — 한 바퀴 돌고 나서 다시 보면 이해됨
4. **각 문서 마지막의 "직접 해보기"를 꼭 해볼 것** — 읽기만으론 안 남음
